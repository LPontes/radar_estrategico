Como a IA Realmente Funciona — sem a magia e sem o medo

O que é um LLM — Large Language Model

❌ Mito

Não é um banco de dados. Não 'sabe' as informações da sua empresa.

✅ Realidade

É um modelo treinado em texto que aprendeu padrões de linguagem e raciocínio.

Um consultor brilhante que leu tudo — mas não conhece sua empresa. Você precisa dar a ele contexto.

O LLM é o cérebro. Os dados da empresa são o mapa. Sem o mapa, o cérebro mais inteligente do mundo não encontra nada relevante.

Por que isso importa?

Ponto-chave para o gestor: não culpe a IA quando ela não sabe do seu negócio — você não deu o mapa.

Redes Neurais — o suficiente para não ter medo

"Não é necessário entender a matemática. É necessário entender a metáfora."

▹

▹

Como um funcionário que após 6 meses antecipa o que o gestor vai pedir

▹

Por que isso importa?

Você não precisa saber construir a rede. Precisa saber o que ela é capaz de fazer pelos seus dados.

O Conceito de Agente: Autonomia e Resolução de Problemas

Da execução passiva à orquestração inteligente de tarefas

"Um Agente é como um estagiário sênior: você passa um objetivo complexo e ele planeja, busca, formata e entrega."

Por que isso importa?

Diferença chave: o LLM responde ao que você pergunta. O Agente age sobre o que você quer alcançar.

Evaluation Harness — Estruturas de Avaliação Rigorosa

Padronizando a medição de performance em sistemas complexos

▹

Framework de software para encapsular, automatizar e padronizar testes em modelos

▹

Ambiente controlado: inputs e outputs medidos sistematicamente

▹

Permite execução de benchmarks (MMLU, HumanEval) para quantificar capacidades

▹

Garante reprodutibilidade científica e comparação transparente entre arquiteturas

"Se o agente é o 'piloto', o Harness é o simulador de voo que certifica que as métricas são estatisticamente válidas."

Por que isso importa?

Sem Harness, confiamos em 'feeling'. Com Harness, confiamos em evidência — a diferença entre achismo e ciência de dados.

Tool Calling — quando a IA deixa de conversar e começa a agir

Sem Tool Calling

LLM só de texto — responde, explica, sugere. Para aí.

Com Tool Calling

LLM + Ferramentas — consulta banco, roda query, chama API, gera gráfico, entrega resultado.

"A diferença entre perguntar 'como fazer uma pesquisa' — e o assistente fazer a pesquisa e trazer o resultado."

É o que torna possível o agent query da Parte Prática desta aula.

Por que isso importa?

Sem Tool Calling, a IA é só texto. Com Tool Calling, ela interroga seus dados.

A Evolução dos Sistemas de Dados

Por que seus sistemas de gestão não foram feitos para análise

OLTP vs OLAP — dois mundos, dois propósitos

Online Transaction Processing

OLTP

  • ▹ ERP, CRM, sistema de gestão, PDV
  • ▹ Otimizado para velocidade de escrita
  • ⚠️ Péssimo para análise — consultas complexas travam o sistema

Exemplo: "Cruzar vendas + clientes + produtos no ERP = sistema lento"

vs

Online Analytical Processing

OLAP

  • ▹ Data Warehouse, cubos de dados
  • ▹ Otimizado para leitura e cruzamento de grandes volumes
  • ✅ Permite perguntas complexas sem travar o transacional

Exemplo: "Receita por região, por categoria, nos últimos 6 meses vs. ano anterior"

A maioria das PMEs tem OLTP mas não tem OLAP. O analista constrói a camada analítica manualmente no Excel. Isso é caro, frágil e não escala.

Por que isso importa?

O gap não é culpa do analista. É um problema de arquitetura — e tem solução.

A Evolução da Arquitetura de Dados

Data Warehouse Clássico

Banco relacional estruturado

  • ✅ Estrutura e governança
  • ❌ Caro, rígido, não aceita dados não estruturados

Data Lake

Armazenamento bruto de tudo

  • ✅ Barato e flexível
  • ❌ Vira um 'pântano de dados' sem governança
⭐

Lakehouse

Combina estrutura + flexibilidade

  • ✅ Parquet/Delta · SQL + Python + Agentes IA
  • ✅ Cloud barata ou local — custo próximo de zero

Por que isso importa?

Uma PME não precisa de infraestrutura de big tech para ter arquitetura moderna. Com DuckDB, você tem um Lakehouse funcional, gratuito, incrivelmente rápido.

BLOCO 3 · 30–35 MINUTOS

Prática

Agent Query ao Vivo — do dado bruto ao Decision Briefing

Sequência da Demonstração ao Vivo

1

Pergunta Simples 5 min

"Qual foi a receita total do último trimestre?"

Recebe o texto, identifica tool SQL, gera a query correta, executa no banco e responde em linguagem natural.

2

Cruzamento 8 min

"Quais são os 5 produtos com maior taxa de abandono de carrinho nos últimos 90 dias?"

Cruza dados de pedidos, produtos e eventos do site. O Tool Calling orquestra múltiplas queries para construir a visão completa.

3

Análise Estratégica 10 min

"Existe correlação entre tempo de entrega e NPS?"

O Agente performa análise estatística, identifica os padrões nos dados cruzados e retorna uma interpretação executiva do impacto.

4

Decision Briefing 7 min

"Qual é a principal alavanca de receita para o próximo trimestre?"

Sintetiza todos os insights descobertos e gera uma recomendação estruturada de uma página, pronta para a mesa da diretoria.

Dataset: DuckDB e-commerce · Agente: LangChain + Tool Calling · Interface: Jupyter Notebook

Por que isso importa?

Este é o Framework EDA em ação: do dado bruto (Extract) à análise (Discover) ao argumento executivo (Act).
